# 60-preview-observability

## Preview

- Preview는 Execution이 생성/종료하는 임시 확인용 실행 자원이다.
- 모니터링 URL과 preview URL은 분리한다.
- URL/TTL/상태는 State에 기록한다.
- 초기 운영 한도: 동시 preview 최대 3개.
- 실패 재시도: 3회(1분/5분/15분), 이후 수동 판단.

## 일반 실행 동시성/스로틀 (B)

- preview 외 일반 실행은 고정 상한 대신 동적 상한으로 운영한다.
- 시작 기본값은 동시 `2`개, 최대 `6`개로 제한한다.
- 상한 조정 신호는 CPU/메모리/큐 대기시간을 사용한다.
- 상한 초과 작업은 드롭하지 않고 대기열에 보관한다.

## Observability

- Observability는 읽기 중심이며 실행 경로를 차단하지 않는다.
- 수집 범위: 로그, 헬스, 이벤트 타임라인.
- 내부 운영 대시보드는 단일 관리자 전용 읽기 채널이다.

## 상태/로그 보관 정책 (B)

- 원본 상태/이벤트/로그는 `90일` 보관한다.
- `90일` 이후에는 요약/압축본으로 `1년` 보관한다.
- 보관 정책 변경은 비용/감사 요구사항 변경 시에만 수행한다.

## 실시간성 기준

- 기본 관측 단위는 하트비트/응답 이벤트다.
- `no_response` 1분 이상: 경고(`warning`)로 기록하고 운영 채널에 알린다.
- `no_response` 3분 이상: 장애(`incident`)로 판정하고 복구 절차로 전환한다.
